# 手机端快速添加优化 PRD

## 目标

让用户在手机添加任务时无需等待分类接口即可选择默认分类，并可在提交前直接标记任务已完成。

## 用户与场景

已有 TaskMaster 同步设置的个人用户，在手机上补录刚完成的事务或快速新建待办。

## 核心功能

- 分类控件首屏展示并默认选中“星标”，同时提供“工作、生活、学习”；云端已有自定义分类返回后保留“星标”并合并实际分类。
- 新增“已完成”勾选项；选中后，任务及完成时间写入后端，并在电脑端同步为已完成任务。

## 交互与视觉

- 保持现有单列表单与紧凑移动端布局。
- 已完成使用标准复选框，放在预计时长之后，避免增加新的操作层级。

## 约束

- 手机页在 Cloudflare Worker 的 `MOBILE_HTML` 中；修改后必须部署 Worker。
- 既有 D1 数据库须先执行增量迁移，不能依赖重建表。
- 桌面端仍以现有任务同步机制导入，避免重复创建任务。

## 参考

参考 TaskMaster 桌面端任务表单的默认分类和完成状态语义。
