## 记录规则

1. 每个发布版本都必须有同名 Git 标签和 GitHub Release。
2. 交付型 Issue 在发布后附上版本/标签并关闭。
3. 未完成的后续工作必须保留在单独的开放 Issue。
4. `CHANGELOG.md` 在同一发布提交中更新；本索引在版本、标签或 Issue 对应关系变化时更新。
