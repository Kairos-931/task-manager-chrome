# feat: 同步机制改进

## 当前同步机制

每次增删改任务都会调用 `persistState()` → `saveData()` → `syncToCloud()`，全量覆盖云端数据。

## 已知问题

1. **全量覆盖，非增量同步** — 每次操作都 POST 全部任务数据，任务多了性能差
2. **同步失败静默** — `syncToCloud` 用 `.catch()` 静默失败，用户不知道同步是否成功
3. **拉取后无反馈** — 云端拉取成功后 UI 直接关闭，用户不知道拉取了多少数据
4. **无冲突检测** — 两端同时修改，后写入的覆盖先写入的，无合并逻辑
5. **chrome.storage.sync 通道残留** — `saveData` 里每次都写 sync，但已标记废弃

## 改进方向

- [ ] 增量同步：只同步变更的 task（按 updatedAt 对比）
- [ ] 同步状态可视化：显示上次同步时间、成功/失败状态
- [ ] 冲突合并策略：按字段粒度合并（如 A 改标题，B 改优先级，两者都保留）
- [ ] 移除 chrome.storage.sync 写入（已废弃）
- [ ] 拉取/上传结果明确反馈（任务数量、耗时）
