## [3.10.0] - 2026-07-13

### 新增

- **增量跨设备同步** — 任务、分类和设置以独立记录同步，服务端使用游标变更日志收敛多端离线操作。
- **删除墓碑** — 已删除任务保留同步记录，旧设备恢复联网后不会复活已删除任务。

### 修复

- **跨设备冲突覆盖** — 不再在冲突后把整份本地快照写回云端；不同任务的离线修改可以同时保留。
- **Worker 安全与手机端完善** — 缺少 `API_TOKEN` 时拒绝服务；分类读取需要认证；PWA 图标可以正常返回。
- **发布版本校验** — 构建后验证源码、包版本和可加载扩展目录版本一致。

## [3.8.0] - 2026-06-17

### 修复（重大）

- **删除任务被恢复 / 完成状态被覆盖** — loadData 不再用云端 merge 覆盖本地
  - 根因：打开插件时总把云端数据 merge 进本地，而 merge 是"并集"语义（云端有、本地无的任务会被加入）。`cloudSyncWrite` 异步推送，关闭插件时可能未完成 → 云端落后、仍含已删任务 → 下次打开 merge 把已删任务"恢复"回来，或用旧状态覆盖本地最新的完成/修改
  - 修复：loadData 改为「本地为权威」——本地有数据时直接用本地，不 merge 云端；仅本地为空（重装/新设备）时从云端恢复
  - 删除整套失效的 merge 函数（mergeStorageData / mergeTasks / mergeCategories）
  - 跨设备拉取改由「从云端拉取」按钮手动触发，避免自动 merge 覆盖本地

## [3.7.0] - 2026-06-16

### 变更（重大）

- **移除 chrome.storage.sync 同步通道** — 国内 Google 不通，实测始终为空且写失败静默误导用户
  - 删除分块读写（loadFromSyncChunked / saveToSyncChunked / loadFromSyncSimple / mergeRemoteData）
  - 删除 sync.ts 的 onChanged 实时监听与 showSyncToast
  - `loadData` 简化为：Cloudflare → 本地备份 → 默认
  - 同步统一走 Cloudflare 单后端，跨设备改为打开时全量拉取

## [3.6.0] - 2026-06-16

### 新增

- **云端数据安全：乐观锁 + 空数据保护** — 防止重装/异常导致空数据反向清空云端
  - 后端 `/api/fullsync`：空数据覆盖非空云端返回 409；`baseUpdatedAt` 过期返回 409 conflict
  - 前端维护云端基准版本，冲突时自动拉取刷新
  - `loadData` 改为云端与本地备份合并，保留未上云的本地改动
  - 「强制上传」按钮走 `force:true` 绕过保护，尊重用户覆盖意图

### 改进

- background.ts：API token 从 URL query 改为 `Authorization` header（避免 token 暴露在 URL/日志）
- task.ts：循环任务双击防抖（500ms），防止重复推进日期

## [1.2.0] - 2026-04-25

### 新增

- **统一存储架构** — 重写 `shared/storage.ts`，采用分块存储（`tm_meta`/`tm_tasks_*`）
  - 三级回退读取：sync 分块 → 旧版 simple key → local 备份
  - 旧版数据自动迁移，升级无感知
- **同步管理面板** — 恢复四宫格同步面板（上传到云端 / 从云端拉取 / 导出文件 / 导入文件）
- **时间戳合并** — 多设备离线冲突时按 `updatedAt` 时间戳取最新版本，两端修改都不丢失
- **数据安全保护** — `loadState` 返回空数据时跳过 `persistState`，防止空默认值覆盖 storage 中的真实数据

### 修复

- 修复 `chrome.d.ts` 类型声明缺失和错误（`onChanged` 位置、`onMessage` 签名）

## [1.0.0] - 2026-04-18

### 新增

- **跨设备数据同步** — `shared/sync.ts`，基于 chrome.storage.sync 实现多设备同步
  - 同步状态指示器（空闲/同步中/已同步/远程更新/错误）
  - 分块存储突破 chrome.storage.sync 单条 8KB 限制
- **双重存储机制** — 改进 `shared/storage.ts`，同时写入 local 和 sync
  - 优先读取 sync 数据，sync 被清空时自动从 local 恢复
  - 双写确保数据安全，防止卸载插件导致数据丢失
- **艾森豪威尔矩阵迁移** — `convert-eisenhower.cjs` 迁移脚本
  - 自动将四象限任务转换为新格式
  - 优先级映射：重要紧急→高、重要不紧急→中
- `chrome-extension-sync/` 可直接加载的扩展目录
- `release/`、`README.md`、`GITHUB_GUIDE.md`、`LICENSE` — 开源发布资源

### 改进

- 安装流程简化，构建系统优化（esbuild 单文件打包）
- 增强导入数据格式验证
